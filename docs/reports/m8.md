## What shipped

**First week (`/dashboard/welcome`).** `lib/queries/onboarding.ts#firstWeekStatus`
derives the three checks from real rows ‚Äî entry (name + title present), primer
(enrolled in `Department.onboardingCourseId` with ‚â•1 lesson completed in that

checks server-side, conditional-stamps `onboardingCompletedAt` on the null

double-click or a race awards once. The rail shows "First week ¬∑ n of 3" while
pending; the Today view lands a new hire on `/dashboard/welcome` on the first
dashboard visit of a session (a session cookie set by `markWelcomeSeen` stops
the repeat) and walls off nothing ‚Äî every route stays reachable.

**The Shadow adapter (stretch, additive).** `lib/shadow/anthropic.ts` calls the
Anthropic Messages API over `fetch` ‚Äî no SDK, so nothing is added to the ¬ß9
allowlist and the file is imported lazily only when `ANTHROPIC_API_KEY` is set.
`lib/shadow/agent.ts#draftBodyFromFacts` is the request-time selector: the
deterministic agent is the default and the test target, the adapter is reached
only with a key, and any failure (including an empty completion) falls back to
the deterministic draft. `requestShadowDraft` was restructured to compose the
body before the transaction, so the one-pending-draft aggregate lock is never
held during the network call. The adapter reads only `ShadowFacts` ‚Äî it never
widens the Shadow's inputs ‚Äî and its system prompt enforces the ¬ß5.7 voice.

**Hardening.** README rewritten for V2 (setup, the five personas, the
simulate:booking loop, the three env placeholders + the 409 mock-failure
convention, the tour). Reduced motion is honored three ways: a global CSS
override collapses every animation/transition (theme crossfade included), and

## Adversarial pass

Four bounded auditors ‚Äî the onboarding flow; the adapter's additive guarantee;
whether the amended ¬ß12 is genuinely covered; a full-product design/voice/
reduced-motion regression ‚Äî each finding adversarially verified. 7 agents, 3
raw findings, 2 confirmed, 1 refuted. **The gate-coverage auditor found
nothing**: every suite the amended ¬ß12 names genuinely exercises its claim. The
two confirmed (both low) were fixed:

- **LOW ‚Äî the completion toast overstated an idempotent re-invoke.**
  `completeOnboarding`'s already-complete branch returns success without
  awarding, but the button toasted "Setup complete ‚Äî 50 XP" on any success.
  The outcome now carries an `awarded` flag and the toast fires only when the
  award actually happened (the once-only XP guard was never in doubt).
- **LOW ‚Äî the adapter could store a blank draft.** A 2xx whitespace-only
  completion passed the emptiness guard before trimming, returning `""` instead
  of falling back. The guard now trims first, so an empty completion throws and
  the deterministic fallback fires.

Refuted: a claimed intermittent seed-integrity failure from the finalize test
"orphaning" an account ‚Äî the file is M7, not M8; `fileParallelism: false` runs
DB suites sequentially and each cleans up in `afterAll`; 97/97 reproduced
across repeated runs, and the only way to fail it is to hand-inject a row.
